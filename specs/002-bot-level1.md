# SPEC-002 · Контракт бота + уровни 0 (случайный) и 1 (жадный)

- **Статус**: approved (в рамках утверждения плана владельцем 2026-07-22)
- **Зависимости**: SPEC-001 (контракт движка)
- **Whitelist файлов**: `site/play/bot.js`, `tests/bot.test.mjs` (создать; другие файлы не трогать)
- **Коммит реализации**: _(заполняется при приёмке)_

## Цель

Единый контракт выбора хода для всех уровней ИИ и два первых уровня: 0 — равновероятный случайный легальный ход (эталон для измерений), 1 — жадная эвристика, уверенно обыгрывающая случайного. Уровень 1 — временный соперник для UI до готовности SPEC-004.

## Не-цели

- Никакого перебора в глубину, альфа-беты, MCTS — это SPEC-004.
- Никакого Web Worker — обёртка появится в SPEC-004.
- Тюнинг силы сверх порога приёмки.

## Контракты

ES-модуль `site/play/bot.js`, единственный экспорт:

```js
chooseMove(s: State, opts: { level: 0 | 1, seed: uint32 }) → Move
```

- `State`/`Move` — из контракта SPEC-001; движок импортируется из `./engine.js`.
- Функция чистая: не мутирует `s` (в т.ч. не трогает `s.rngState`); вся случайность бота — собственный mulberry32 от `opts.seed`.
- Неизвестный `level` → Error `.code='BAD_LEVEL'`. Вызов при `phase='over'` → Error `.code='GAME_OVER'`.

## Функциональные требования

- **FR-1** Возвращаемый ход всегда ∈ `legalMoves(s)` (для обоих уровней, в любом состоянии).
- **FR-2** Детерминизм: одинаковые `(s, opts)` → одинаковый ход.
- **FR-3** Уровень 0: равновероятный выбор из `legalMoves` ГПСЧ от `opts.seed`.
- **FR-4** Уровень 1 оценивает каждый легальный ход суммой (веса зашиты константами в начале файла):
  немедленный прирост заполнения линии с учётом близости к завершению; ожидаемые очки клетки стены, куда линия выложится (по текущей стене); штрафные очки за плитки, уходящие на пол этим ходом (по реальной таблице пола); небольшой штраф за взятие жетона при почти полном поле пола и небольшой бонус за жетон в ранней части раунда.
- **FR-5** Уровень 1 при прочих равных не выбирает `dest:'floor'`, если существует ход без переполнения пола (тестируемый сценарий).
- **FR-6** Равные оценки уровня 1 разрешаются ГПСЧ от `opts.seed` (не первым по порядку).

## Инварианты

- **INV-1** `chooseMove` не изменяет переданное состояние (побайтное сравнение JSON до/после).

## Критерии приёмки

| Что        | Проверка                                                                                                    |
| ---------- | ----------------------------------------------------------------------------------------------------------- |
| FR-1, FR-2 | Фаззинг: 200 партий level1 vs level0, каждый ход проверяется на принадлежность `legalMoves`                 |
| FR-3       | Хи-квадрат-лайт: на фиксированном состоянии 1000 вызовов с разными сидами покрывают все легальные ходы      |
| FR-4, FR-5 | Точечные сценарии с известным лучшим ходом                                                                  |
| Сила       | level1 выигрывает у level0 ≥ 90% из 200 партий (сиды 1..200, старт по очереди), тест в `tests/bot.test.mjs` |
| INV-1      | Юнит-тест                                                                                                   |

## Тест-план (`tests/bot.test.mjs`, node:test)

1. Легальность и детерминизм обоих уровней (фаззинг-цикл).
2. Level 0: покрытие всех легальных ходов по сидам.
3. Level 1: сценарий «не сливай в пол без нужды» (FR-5).
4. Level 1: сценарий «завершение линии ценнее пустой траты» (FR-4).
5. Винрейт level1 vs level0 ≥ 90% на 200 партиях.
6. Иммутабельность состояния.
7. Ошибки BAD_LEVEL и GAME_OVER.

## Бюджеты

- Ход уровня 1 ≤ 5 мс; 200 тестовых партий ≤ 30 с суммарно с фаззингом.

## Запрещено

- Менять файлы вне whitelist (в т.ч. `site/play/engine.js` — если движка не хватает, это поправка к SPEC-001 через оркестратора, а не правка движка).
- Зависимости, DOM, `Math.random`, `Date.now`.

## Amendments

### A-1 (2026-07-22) · FR-6 добавлен в приёмку

**Причина.** FR-6 (тай-брейк равных оценок через ГПСЧ, а не первым по порядку) был сформулирован, но отсутствовал в таблице «Критерии приёмки» и «Тест-плане» — пробел спеки, найден ревьюером.

**Дополнение к критериям приёмки и тест-плану.** Тест 8: симметричное тай-состояние (например, две фабрики по одной плитке разных цветов при одинаковом контексте стены/линий → равные оценки уровня 1); на выборке сидов выбор распределяется между связанными ходами (не всегда первый по порядку `legalMoves`) и зависит от `opts.seed`.
