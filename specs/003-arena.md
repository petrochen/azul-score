# SPEC-003 · Турнирный стенд (arena)

- **Статус**: approved (в рамках утверждения плана владельцем 2026-07-22)
- **Зависимости**: SPEC-001, SPEC-002
- **Whitelist файлов**: `tools/arena.mjs`, `tests/arena.test.mjs` (создать; другие файлы не трогать)
- **Коммит реализации**: _(заполняется при приёмке)_

## Цель

Измерительный прибор силы ботов: CLI для массовых партий бот-против-бота с воспроизводимой статистикой. Без него приёмка SPEC-004 и любой тюнинг эвристик недоказуемы.

## Не-цели

- Никакого тюнинга весов и новых уровней ботов — стенд только меряет.
- Никакой параллельности через worker_threads (если 500 партий укладываются в бюджет однопоточно).

## Контракты

CLI: `node tools/arena.mjs --a <level> --b <level> [-n 500] [--seed 42] [--verbose]`

- `<level>` — целое, передаётся в `chooseMove` из SPEC-002 как `opts.level` (стенд не знает списка уровней — валидирует сам бот).
- Партия `i` (0-based): сид партии = `seed + i`; стартует игрок `i % 2` (снятие преимущества первого хода); сиды ботов производны от сида партии и роли (A/B), а не от игрока.
- Вывод (stdout, финальный блок обязательно в этом формате, по строке на метрику):

```
games: 500
A wins: 462 (92.4%)  B wins: 33 (6.6%)  ties: 5 (1.0%)
A winrate 95% CI: [89.7%, 94.4%]
avg score A: 54.3  avg score B: 31.8
avg rounds: 5.9
elapsed: 12.4s
```

- Также экспорт для тестов: `runMatch({a, b, n, seed}) → {aWins, bWins, ties, avgScoreA, avgScoreB, avgRounds}` (тот же код, что CLI).

## Функциональные требования

- **FR-1** Детерминизм: два запуска с одинаковыми аргументами дают идентичный вывод (кроме `elapsed`).
- **FR-2** Каждый ход каждого бота проверяется на принадлежность `legalMoves`; нарушение — немедленное падение с описанием (номер партии, сид, ход).
- **FR-3** Партия ведётся строго через контракт движка (`legalMoves`/`applyMove`/`isDraftOver`/`resolveRound`) до `phase='over'`; защита от зацикливания — лимит 50 раундов с падением.
- **FR-4** 95% CI — интервал Уилсона по винрейту A (ничьи — не победы).
- **FR-5** `--verbose` печатает строку на партию: номер, сид, кто начал, счёт, раунды, победитель.
- **FR-6** Неизвестные аргументы/уровни → понятная ошибка и exit code 1.

## Инварианты

- **INV-1** `totalTiles === 100` после каждой партии (выборочно: проверять финальное состояние каждой партии).

## Критерии приёмки

| Что        | Проверка                                                                                |
| ---------- | --------------------------------------------------------------------------------------- |
| FR-1       | Тест: два вызова `runMatch` с одним сидом → deepEqual результатов                       |
| FR-2, FR-3 | Фаззинг внутри теста: 50 партий level1 vs level0 без падений                            |
| FR-4       | Юнит-тест интервала Уилсона на известных значениях                                      |
| Смоук CLI  | Тест запускает CLI через child_process на 20 партий и парсит финальный блок             |
| Сила L1    | `runMatch(level1 vs level0, n=200, seed=1)` → винрейт A ≥ 90% (дубль-контроль SPEC-002) |

## Тест-план (`tests/arena.test.mjs`, node:test)

1. Детерминизм runMatch.
2. 50 партий без нарушений легальности и инвариантов.
3. Интервал Уилсона: 3 контрольные точки.
4. CLI-смоук через child_process.
5. Винрейт level1 vs level0 ≥ 90% (200 партий).

## Бюджеты

- 500 партий level1 vs level0 ≤ 60 с однопоточно.
