# SPEC-001 · Движок правил Azul (2 игрока)

- **Статус**: approved (в рамках утверждения плана владельцем 2026-07-22)
- **Зависимости**: нет
- **Whitelist файлов**: `site/play/engine.js`, `tests/engine.test.mjs` (создать; другие файлы не трогать)
- **Коммит реализации**: _(заполняется при приёмке)_

## Цель

Полная, детерминированная, тестируемая модель партии Azul на двух игроков в виде чистых функций над plain-JSON состоянием. Движок — фундамент для бота (SPEC-002/004), турнирного стенда (SPEC-003) и UI (SPEC-005); работает и в браузере (ES-модуль), и в node.

## Не-цели

- Никакого UI, DOM, Web Worker — только логика.
- Никакого ИИ и оценочных функций.
- Партии на 3–4 игроков (модель не должна этому мешать, но реализация и тесты — только для 2).
- Вариант «серой стороны» планшета (свободная раскладка) — вне scope.

## Контракты

ES-модуль `site/play/engine.js`, named exports. Все функции чистые: вход не мутируется, возвращается новое состояние. Никаких `Math.random`/`Date.now` — только ГПСЧ из состояния.

### Кодировки

- Цвета: `0` синий, `1` жёлтый, `2` красный, `3` чёрный, `4` бирюзовый.
- Стена: клетка ряда `r` для цвета `k` — колонка `(r + k) % 5` (как на основной странице сайта).
- ГПСЧ: mulberry32; его 32-битное состояние хранится в `State.rngState` и обновляется при каждом использовании.

### Состояние

```js
State = {
  rngState: uint32,
  round: int,                 // с 1
  phase: 'draft' | 'over',
  current: 0 | 1,             // чей ход в фазе набора
  startPlayer: 0 | 1,         // кто начнёт следующий раунд
  firstTakenBy: null | 0 | 1, // кто первым взял из центра в этом раунде
  factories: number[5][5],    // factories[f][color] = число плиток
  center: number[5],          // center[color]
  bag: number[5],
  discard: number[5],
  players: [Player, Player],
  result: null | { winner: 0 | 1 | 'tie',
                   breakdown: [{rows, cols, colors, bonus}, {…}] },
}
Player = {
  lines: { color: int | null, count: int }[5], // вместимость ряда i = i+1
  wall: boolean[5][5],                          // wall[r][c]
  floorTiles: int,                              // плитки пола, БЕЗ жетона
  hasToken: boolean,                            // жетон первого игрока на полу
  score: int,
}
Move = { source: 'factory', factory: 0..4, color: 0..4, dest: 0..4 | 'floor' }
     | { source: 'center',               color: 0..4, dest: 0..4 | 'floor' }
```

### API

```js
newGame(seed: uint32, opts?: { startPlayer?: 0|1 }) → State
  // мешок 5×20, 5 фабрик по 4 плитки уже разложены, center пуст,
  // phase='draft', current = startPlayer (по умолчанию 0)

legalMoves(s: State) → Move[]        // [] тогда и только тогда, когда phase==='over'
applyMove(s: State, m: Move) → State // Error с полем .code='ILLEGAL_MOVE' на нелегальный ход
isDraftOver(s: State) → boolean      // фабрики и центр пусты
resolveRound(s: State) → { state: State, log: RoundLog }
  // только при isDraftOver, иначе Error .code='DRAFT_NOT_OVER'
totalTiles(s: State) → number        // сумма плиток по всем зонам (для инвариант-тестов)

RoundLog = {
  placements: { player, row, col, color, pts }[],   // в порядке применения
  floors:     { player, tiles, penalty }[],          // tiles — занятые ячейки пола (с жетоном)
  gameOver: boolean,
  bonuses?: [{ rows, cols, colors, bonus }, { … }],  // при gameOver
  winner?: 0 | 1 | 'tie',
}
```

## Функциональные требования

- **FR-1** `newGame`: 100 плиток (по 20 цвета), 20 на фабриках, 80 в мешке; раскладка детерминирована сидом (один сид → побайтно одинаковое состояние).
- **FR-2** Взятие с фабрики: игрок получает все плитки выбранного цвета этой фабрики (цвет там должен присутствовать), остальные плитки фабрики перемещаются в центр.
- **FR-3** Взятие из центра: игрок получает все плитки цвета из центра; если `firstTakenBy === null`, игрок получает жетон (`hasToken=true`, `firstTakenBy=игрок`).
- **FR-4** Размещение в линию `d`: легально только если линия пуста или её цвет совпадает, линия не полна, и на стене в ряду `d` нет плитки этого цвета. Плитки докладываются до вместимости `d+1`, излишек идёт на пол.
- **FR-5** `dest:'floor'` легален всегда; все взятые плитки идут на пол.
- **FR-6** Пол: занятых ячеек (плитки + жетон) не больше 7; плитки сверх — сразу в `discard`.
- **FR-7** После `applyMove` ход переходит другому игроку; `legalMoves` перечисляет все и только легальные ходы текущего игрока.
- **FR-8** `resolveRound` — выкладка: для каждой ПОЛНОЙ линии (ряды сверху вниз, игрок 0 затем игрок 1) одна плитка идёт в клетку стены своего цвета, остальные `count-1` — в `discard`; неполные линии не меняются.
- **FR-9** Очки плитки при выкладке: длина горизонтальной цепочки (если ≥2) + длина вертикальной (если ≥2); одиночная — 1. Считается в момент выкладки с учётом уже выложенных в этом же `resolveRound` плиток.
- **FR-10** Штраф пола по занятым ячейкам: 0,−1,−2,−4,−6,−8,−11,−14 для 0..7; счёт не опускается ниже 0; после вычета `floorTiles→discard`, `floorTiles=0`, `hasToken=false`.
- **FR-11** Жетон: `startPlayer` следующего раунда = `firstTakenBy` (если никто не брал из центра — прежний `startPlayer`); в новом раунде `firstTakenBy=null`, `current=startPlayer`.
- **FR-12** Конец партии: если после выкладки у кого-то есть полный ряд стены — бонусы (ряд +2, столбец +7, цвет +10), `phase='over'`, `result.winner` (больше очков; при равенстве больше полных рядов; иначе `'tie'`).
- **FR-13** Продолжение партии: фабрики заполняются по 4 из мешка; пустой мешок пополняется из `discard`; если плиток на все фабрики не хватает — заполняется сколько есть; если после заполнения фабрики и центр пусты — партия завершается как в FR-12 (защита от вырожденного случая).
- **FR-14** Иммутабельность: `applyMove`/`resolveRound` не мутируют аргументы (глубокое сравнение входа до/после).
- **FR-15** `RoundLog` полностью описывает произошедшее (каждая выложенная плитка с очками, штрафы обоих игроков) — этого достаточно UI для анимации без пересчёта.

## Инварианты

- **INV-1** `totalTiles(s) === 100` после любой операции.
- **INV-2** Цвет линии `d` никогда не совпадает с цветом, уже стоящим на стене ряда `d`.
- **INV-3** `score ≥ 0` у обоих игроков в любом состоянии.
- **INV-4** `floorTiles + hasToken ≤ 7`.
- **INV-5** На стене в каждом ряду и каждом столбце каждый цвет встречается не более 1 раза (следует из раскладки, но проверяется).

## Тест-план (обязательные тесты, `tests/engine.test.mjs`, node:test)

1. newGame: суммы плиток, детерминизм сида.
2. Взятие с фабрики: перенос остатка в центр.
3. Взятие из центра: жетон первому, второму — нет.
4. Легальность линий: чужой цвет, полная линия, цвет уже на стене, переполнение в пол.
5. Пол: переполнение сверх 7 → discard; жетон занимает ячейку.
6. resolveRound: очки одиночной плитки =1; горизонтальная цепочка; крест h+v; порядок «сверху вниз» влияет на очки (сценарий с двумя линиями одного столбца).
7. Штрафы: таблица 1..7, кламп на 0.
8. Жетон → startPlayer следующего раунда; никто не брал из центра → прежний.
9. Пополнение мешка из discard.
10. Конец партии: полный ряд, бонусы, победитель, тай-брейк по рядам, ничья.
11. Иммутабельность.
12. Фаззинг 1000 партий: инварианты, завершаемость, легальность всех ходов.
