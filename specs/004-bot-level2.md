## Цель

Основной соперник игры: уровень 2 — полный перебор с альфа-бета-отсечением до конца текущего раунда (внутри раунда игра детерминирована и с открытой информацией) и оценкой позиции на границе раунда. Плюс обёртка Web Worker, чтобы UI не замерзал.

## Не-цели

- Просмотр через границу раунда (случайный добор) — это MCTS, уровень 3, V2.
- Автотюнинг весов оптимизатором — веса подбираются вручную по прогонам стенда.
- Уровни сложности между 1 и 2 (ослабления) — V2.

## Контракты

- `chooseMove(s, {level: 2, seed, timeMs?})` — тот же экспорт из `site/play/bot.js`, что в SPEC-002. `timeMs` (по умолчанию 1000): мягкий бюджет — итеративное углубление, возврат лучшего хода последней завершённой глубины.
- `site/play/bot-worker.js` — классический Worker: принимает `{id, state, opts}` постом, отвечает `{id, move}`; ошибки — `{id, error: {code, message}}`. Никакой логики в worker-обёртке — только вызов `chooseMove`.
- Оценочная функция на границе раунда (веса — именованные константы в начале файла): реальные очки после гипотетического `resolveRound` + потенциал незавершённых линий (близость к выкладке, ожидаемые очки клетки) + прогресс столбцов/цветов к бонусам + ценность жетона первого игрока − ожидаемые будущие штрафы. Оценка симметричная: score(me) − score(opp).

## Функциональные требования

- **FR-1** Ходы всегда ∈ `legalMoves`; детерминизм при одинаковых `(s, opts)` (тай-брейк равных оценок — ГПСЧ от `opts.seed`).
- **FR-2** Поиск: минимакс с альфа-бета по ходам обоих игроков до `isDraftOver`, затем оценка через гипотетический `resolveRound` (без мутации входа).
- **FR-3** Итеративное углубление с бюджетом `timeMs`; при истечении — лучший ход последней завершённой глубины; глубина ≥ 2 гарантирована всегда.
- **FR-4** Упорядочивание ходов (например, по эвристике уровня 1) — обязательное, иначе отсечение не работает.
- **FR-5** Worker-обёртка: корректная сериализация, соответствие `id` запрос-ответ, ошибка BAD_LEVEL/GAME_OVER пробрасывается.
- **FR-6** `bot.js` остаётся чистым модулем без DOM/Worker API (worker-обёртка — отдельный файл).

## Инварианты

- **INV-1** `chooseMove` не мутирует состояние.
- **INV-2** Поведение уровней 0 и 1 не изменилось (их тесты из SPEC-002 проходят без правок).

## Тест-план (`tests/bot2.test.mjs`)

1. Легальность и детерминизм L2 (фаззинг 50 партий).
2. Тактический сценарий: L2 находит ход, выигрывающий очки через два хода (подготовленная позиция, где жадный L1 ошибается).
3. Бюджет времени: 50 мс → ход есть; сравнение с 5000 мс.
4. Иммутабельность.
5. Контракт worker-сообщений.
6. Регресс: tests/bot.test.mjs зелёный без правок.

## Бюджеты

- Средний ход ≤ 1.5 с при timeMs=1000 (реальный бюджет соблюдать жёстче: p95 ≤ 2 с).
- Прогон 500 партий L2 vs L1 на стенде ≤ 15 мин (для приёмки допустимо -n 200, если дольше).
