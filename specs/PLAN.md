# Мастер-план: игра Azul против бота

Статус: approved (владельцем 2026-07-22) · Владелец: @petrochen · Процесс: [specs/README.md](README.md)

## Что строим

Отдельная desktop-страница **azul.petrochenko.info/play**: партия Azul 1-на-1 против бота по полным правилам. Сильный соперник (альфа-бета до конца раунда), графика — керамика, сгенерированная Nano Banana на этапе сборки. Существующий инструмент счёта не затрагивается.

## Архитектура

```
site/play/
  index.html        — страница игры (desktop ≥1000px; уже — заглушка «откройте на компьютере»)
  engine.js         — движок правил: чистые функции, ES-модуль, работает и в браузере, и в node
  bot.js            — ИИ: общий контракт chooseMove(), уровни 1..2
  bot-worker.js     — обёртка Web Worker вокруг bot.js
  ui.js             — рендер и взаимодействие
  assets/           — PNG от Nano Banana (плитки, тарелка, фон, аватар)
tools/
  arena.mjs         — турнирный стенд bot-vs-bot (node CLI)
scripts/
  gen_assets.py     — генерация графики (по образцу AgentPaymentsVisa/gen_catalog_images.py)
tests/
  engine.test.mjs   — юнит-тесты движка
  bot.test.mjs      — тесты контракта бота
```

Ключевые решения:

- **Движок отделён от UI и от бота** — он же потом бесплатно даёт «подсказку хода» в живом счёте и разбор партий.
- **Вся случайность через seed** (mulberry32): партия воспроизводима по номеру, баги ботов отлаживаемы, стенд детерминирован.
- **Бот в Web Worker** — интерфейс не замерзает; тайм-бюджет хода фиксирован.
- Никаких сборщиков и зависимостей: ES-модули как есть, тесты `node --test`.

## Декомпозиция на спецификации

### SPEC-001 · Движок правил

Полная модель партии 2 игроков: фабрики (5 для двоих), центр, мешок/сброс, линии узора, стена, пол, жетон первого игрока, фазы раунда, конец партии, тай-брейк.
**Контракт (ядро):** `newGame(seed) → State`, `legalMoves(State) → Move[]`, `applyMove(State, Move) → State`, `resolveRound(State) → {State, log}`, `isGameOver(State)`, `finalScore(State)`. Состояние — plain JSON, `Move = {source: 'factory'|'center', factoryIdx?, color, dest: 0..4|'floor'}`.
**Ключевые инварианты:** учёт всех 100 плиток по зонам в любом состоянии; запрет цвета в линии при занятой клетке стены; счёт ≥ 0.
**Приёмка:** юнит-тест на каждое правило из вкладки «Правила игры» (включая «частые ошибки»), золотая партия с известным итогом, 1000 случайных само-партий без нарушения инвариантов.
Зависимости: нет. Оценка: 1 сессия субагента.

### SPEC-002 · Контракт бота + уровень 1

Единый интерфейс всех уровней: `chooseMove(State, {level, timeMs, seed}) → Move`. Уровень 1 — жадная эвристика: немедленные очки + грубый потенциал − ожидаемый штраф пола.
**Приёмка:** винрейт ≥ 90% против случайного бота на 200 партиях; любой возвращённый ход ∈ `legalMoves`.
Зависимости: SPEC-001. Оценка: полсессии.

### SPEC-003 · Турнирный стенд

`node tools/arena.mjs --a level1 --b random -n 500 --seed 42`: параллельные партии, винрейт с доверительным интервалом, средние счета, распределение длины партий, полная воспроизводимость по сиду. Это измерительный прибор — без него сила ботов дальше не доказуема.
**Приёмка:** два запуска с одним сидом дают идентичный результат; 500 партий ≤ 1 мин.
Зависимости: SPEC-001, SPEC-002. Оценка: полсессии (той же, что 002).

### SPEC-004 · Бот уровня 2 (основной соперник)

Альфа-бета до конца раунда (внутри раунда игра детерминирована) с итеративным углублением под тайм-бюджет, оценка позиции на границе раунда: набранные очки, потенциал рядов/столбцов/цветов, риск пола, ценность жетона первого игрока. Веса эвристики тюнингуются стендом SPEC-003 (сотни партий против уровня 1 и против вариантов самого себя). Обёртка Web Worker.
**Приёмка:** винрейт ≥ 70% против уровня 1 на 500 партиях; ход ≤ 1.5 с; UI-поток не блокируется.
Зависимости: SPEC-001..003. Оценка: 1 сессия с тюнингом.

### SPEC-005 · UI игры (desktop)

Раскладка: ряд фабрик + центр сверху, планшет игрока и планшет бота рядом. Взаимодействие: клик по группе плиток на фабрике/в центре → подсветка легальных линий → клик по линии/полу. Анимация переноса, ход бота с короткой паузой и подсветкой, лог партии, экраны конца раунда и конца партии (с разбором бонусов), выбор уровня и кто ходит первым, «новая партия». При ширине <1000px — заглушка со ссылкой на основной инструмент.
**Приёмка:** Playwright-сценарий полной партии против уровня 1 без ошибок консоли; ручная приёмка владельцем на превью-URL.
Зависимости: SPEC-001 (контракт), SPEC-002 (играбельно уже с уровнем 1). Параллелится с SPEC-004. Оценка: 1 сессия.

### SPEC-006 · Графика (Nano Banana)

`scripts/gen_assets.py` по образцу `AgentPaymentsVisa/scripts/gen_catalog_images.py`: ключ из Keychain, модель `gemini-2.5-flash-image`. Ассеты: спрайт 5 плиток одним изображением (единый стиль, нарезка скриптом), тарелка-фабрика, текстура стола, аватар бота, OG-картинка. UI подключает PNG с CSS-фолбэком (текущие градиенты), так что спека не блокирует ничего.
**Приёмка:** плитки различимы при 60px; владелец утверждает вид на превью; в репо — и скрипт, и результат.
Зависимости: нет (параллелится со всем). Оценка: полсессии итераций промптов.

### SPEC-007 · Интеграция и релиз

Кнопка «Играть с ботом 🖥» на главной странице (видна на десктопе), деплой `/play` в прод, обновление README (раздел про игру и архитектуру), обновление памяти проекта, финальный смоук обеих страниц.
**Приёмка:** прод-смоук Playwright: обе страницы без ошибок, счётчик-инструмент не изменился ни на байт, кроме кнопки.
Зависимости: SPEC-004, 005, 006. Оценка: полсессии.

## Порядок и параллельность

```
SPEC-001 ──► SPEC-002 ──► SPEC-003 ──► SPEC-004 ──┐
   │                                              ├──► SPEC-007
   └────────► SPEC-005 (после контракта 001) ─────┤
SPEC-006 (в любой момент) ────────────────────────┘
```

По сессиям: **(1)** 001+002+003 → уже можно играть в консоли; **(2)** 004 (тюнинг стендом) параллельно с запуском субагента на 005; **(3)** приёмка 005, 006, 007 → релиз. Итого ~3 рабочие сессии.

## Ворота качества (сквозные)

- Все тесты всех реализованных спек зелёные перед каждым коммитом (`node --test`).
- Ходы бота любого уровня всегда ∈ `legalMoves` (проверяется стендом на каждом прогоне).
- Ни одного изменения в `site/index.html` до SPEC-007, и там — только кнопка.
- Прод-деплой только после ручной приёмки владельцем превью-URL.
